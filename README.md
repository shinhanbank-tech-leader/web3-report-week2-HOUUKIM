## 1. 서론

2009년 비트코인의 등장은 기존 금융 지급결제 패러다임에 근본적인 물음을 던졌다. 은행도 카드사도 없이, 수학적 알고리즘과 분산 네트워크만으로 가치를 이전할 수 있다는 개념은 수백 년간 중앙화된 구조로 운영되어 온 금융 인프라에 대한 도전이다. 본 보고서는 두 시스템의 구조적 차이를 금융권 실무 관점에서 비교 분석한다.

---

## 2. 기존 금융 지급결제 구조

국내 금융 지급결제는 금융결제원(소액), 한국은행 BOK-Wire+(거액), 한국예탁결제원(증권) 등 중앙화된 인프라를 축으로 운영된다. 카드 결제 한 건에도 고객 → VAN사 → 카드사 → 은행 → 금융결제원으로 이어지는 다중 중개 구조가 작동하며, 실제 정산은 D+1~3 영업일 후 이루어진다.

신뢰의 근거는 전자금융거래법·여신전문금융업법 등 법적 강제력과 금융감독원의 제도적 감독이다. 기관 자체가 아닌 기관을 규율하는 제도를 신뢰하는 **'위임된 신뢰'** 모델이다. 오류·사기 발생 시 기관이 개입하여 취소·환불이 가능하다는 점이 소비자 보호 측면의 핵심 강점이다.

## 3. 비트코인 결제 구조

비트코인은 **UTXO(Unspent Transaction Output)** 모델 기반으로 자산을 관리한다. 계좌 잔액 대신 '아직 사용되지 않은 출력값 덩어리'로 자산이 표현되며, 한번 사용된 UTXO는 재사용이 불가능하여 이중 지불(Double Spend)을 원천 차단한다. Input 합계와 Output 합계의 차액은 채굴자 수수료(Transaction Fee)로 귀속된다.

소유권 증명은 **ECDSA(타원곡선 디지털 서명)** 방식으로 이루어진다. 개인키로 거래에 서명하면, 서버는 서명에서 공개키를 수학적으로 복원하여 자산 보유 여부를 검증한다. 개인키를 직접 전송하지 않고 서명만으로 소유권을 증명하는 구조가 기존 인증 방식과의 근본적 차이다.

## 4. 구조적 차이 비교

### 4.1 핵심 항목 비교표

| 비교 항목 | 기존 금융 지급결제 | 비트코인 결제 |
|-----------|-----------------|-------------|
| **결제 구조** | 중앙화 (은행·카드사 중개) | 탈중앙화 (P2P 직접 전송) |
| **거래 검증** | 중앙 원장 (금융기관) | 분산 노드 합의 (PoW) |
| **정산 시간** | D+1~3일 (국제송금 최대 5일) | 약 10분 (1블록 기준) |
| **수수료** | 카드 1.5~3%, 송금 건당 수만 원 | 데이터 크기 기준 가변 |
| **운영 시간** | 영업시간 (주말·공휴일 제한) | 24시간 365일 |
| **취소·환불** | 가능 (기관 중재) | 불가 (불변성) |
| **규제** | 완전 편입 (전자금융거래법) | 진행 중 (국가별 상이) |

### 4.2 4가지 구조적 차이

**1 원장의 형태: Account vs UTXO**

| | 기존 금융 | 비트코인 |
|--|---------|--------|
| 방식 | 상태 업데이트 (Update) | 이벤트 기록 (Insert) |
| 구조 | DB 잔액 필드 갱신 | UTXO 수표 파괴 후 새 수표 생성 |

**2 신뢰의 주체: 기관 vs 알고리즘**

- **기존 금융**: 중앙 서버(중앙은행, 시중은행)가 장부의 정답을 결정. FEP 전문이 신뢰받는 이유는 '은행'이라는 공인 기관이 보증하기 때문
- **비트코인**: 나카모토 컨센서스(PoW + 최장 체인 원칙)라는 수학적 규칙에 따라 "가장 많은 연산이 들어간 장부"를 정답으로 자동 합의

**3 거래 처리: 승인/정산 분리 vs 즉시 결제**

- **기존 금융**: FEP 승인(실시간처럼 보임) + BOK-Wire 차액 정산(며칠 뒤) — 메시지 전송과 자금 이동이 분리
- **비트코인**: 블록에 기록되는 순간이 곧 자금 이동 — 별도 정산 불필요 (RTGS 성격)

**4 무결성 보장: 방화벽 vs 경제적 비용**

- **기존 금융**: 허가된 사용자만 접속 가능한 폐쇄망(Closed Network) + 방화벽으로 조작 차단
- **비트코인**: 누구나 접속 가능한 개방망(Open Network). 데이터 조작에 천문학적 전기료가 드는 경제적 장벽(PoW)으로 보안 확보

## 5. 결론

두 시스템은 '돈을 어떻게 기록하느냐'부터 근본적으로 다르다. 기존 금융은 DB의 잔액 필드를 갱신하는 상태 업데이트(Update) 방식인 반면, 비트코인은 UTXO라는 수표 뭉치를 파괴하고 새로 생성하는 이벤트 기록(Insert) 방식이다. FEP 전문이 신뢰받는 이유는 은행이라는 공인 기관이 보증하기 때문이지만, 비트코인은 나카모토 컨센서스라는 수학적 규칙이 정답을 자동 합의한다. 기존 금융은 FEP 승인과 BOK-Wire 차액 정산이 분리되어 있지만, 비트코인은 블록에 기록되는 순간이 곧 자금 이동이다.

**✅ 비트코인 결제의 장점**
- **즉시성**: 국경 없이 24시간 운영. 환거래 은행을 거치는 기존 국제 송금 대비 비용·속도 면에서 구조적 우위
- **탈중앙화**: 단일 장애점 없음. 검열 저항성이 높아 특정 기관의 판단으로 거래가 차단되지 않음

**❌ 비트코인 결제의 단점**
- **금융사고에 취약**: 거래 취소가 불가능해 실수나 사기 피해 시 구제 수단이 없음. 개인키를 분실하면 자산이 영구 소멸
- **가격 변동성**: 결제 수단으로 사용하기에 가치가 불안정

두 시스템은 대체재가 아닌 보완재로 발전할 가능성이 높다. 기존 금융의 법적 안정성·소비자 보호와 비트코인의 탈중앙성·즉시 결제를 결합한 CBDC, 스테이블코인 등이 현실적 진화 방향이다.
